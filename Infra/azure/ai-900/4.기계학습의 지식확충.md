# 1. 특징량(特徴量)
- 모델이 출력하는 결과에 영향을 주는 수치 데이터
- 연령 성별 직업 수입 혈압 체중 신장 등
- ex: 택시의 운임을 에측하는 모델의 특징량은 [이동 거리, 이동 시간] 등이 된다.

# 2. 특징량 엔지니어링
- 모델의 퍼포먼스를 최대화하기 위해 데이터의 특징량을 선택, 변환하는 프로세스
- 모델을 작성할 때 **새로운 특징량을 생성**하려고 하면 특징량 엔지니어링이 필요하다
- 가능한 심플한 모델로 정밀도(精度)가 높은 예측을 할 수 있으면 좋다
    - 특징량의 선택 : 불필요한 특징량을 삭제하는 것으로 과학습을 막는다
    - 특징량의 변환 : (정규화와 표준화) 너무 많은 표본에 대해선 특징량의 규모를 조정
    - 변수 변환 : 로그변환이나 역수변환 등 특징량의 분배를 변경
    - 카테고리컬(범주형) 데이터의 처리 : 문자열이나 카테고리컬 데이터를 수치값으로 변환
    - 결손치의 처리 : 결손치의 삭제 혹은 평균치나 중앙값으로 보완
    - 특징량의 차원삭감 : 고차원 데이터 셋을 더 취급하기 쉽도록 저차원 형식으로 변환

# 3. 과학습
- 데이터에 편향(偏り)이 있는 경우, 그 편향(偏り)에만 적합한 모델이 만들어지는 현상
- 모델이 훈련 데이터에 너무 적합하게 만들어져서 새로운 데이터에 대한 예측력이 떨어지는 현상

# 4. 데이터 분할
- 훈련용, 검증용, 평가용으로 분할
- 일반적인 분할 방법
    - 무작위 분할 : 데이터셋을 무작위로 섞어서 훈련용, 검증용, 평가용으로 나누는 방법
- 훈련용
    - 모델의 학습을 위해 사용
- 검증용
    - 모델의 튜닝을 위해 사용
    - 훈련용 데이터로 훈련을 계속하면서 훈련용에만 적합한 모델이 되지 않도록 검증용 데이터로 모델의 방향성을 교정
- 평가용
    - 모델의 평가를 위해 사용
