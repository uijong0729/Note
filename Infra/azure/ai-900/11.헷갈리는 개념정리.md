1. 공평성과 포괄성
    - 공평성 : AI 판단이 치우치지 않도록 
    - 포괄성 : 보다 많은 유저를 확보하도록

2. 어카운트빌리티와 투명성
    - 어카운트빌리티 : 책임의 소재
    - 투명성 : 설명

3. 분류와 회귀
    - 분류
        - 返済できるかどうか予測する (된다vs안된다로 분류)
        - 평가지표
            - 정확도
            - 정밀도
            - 재현율 (ex : TPR(真陽性率))
            - F1 스코어 (정밀도와 재현율의 조화평균)
    - 회귀
        - 不動産価格を予測する (구체적인 값을 예측)
        - 평가지표
            - RMSE（二乗平均平方根誤差）: 평균제곱근오차
            - MAE（平均絶対誤差）: 평균절대오차
            - R²（決定係数）: 결정계수

4. 분류와 클러스터링
    - 분류 
        - 정답 라벨이 있고, 어떤 분류에 속할지 예측하는 것이 목적
        - 예 : 호흡기 질환 리스크를 경도, 중도로 분류한다
    - 클러스터링
        - 정답 라벨이 없고, 주어진 정보에서 어떻게든 그룹을 나누는 것이 목적
        - RFM분석(Recency、Frequency、Monetary)
        - 예 : 고객의 구매 이력과 방문 패턴을 바탕으로 비슷한 성향의 고객 그룹을 자동으로 찾는다

5. Azure AI Vision vs Azure AI Custom Vision
    - Azure AI Vision : 동영상 분석 가능
    - Azure AI Customer Vision : 동영상 분석 불가
    - Azure AI Video Indexer : 동영상 전용 분석 서비스

6. 자동기계학습 (自動機械学習, AutoML)
- 수동으로 설정해줘야 하는 것
    - 라벨
    - 트레이닝 데이터
    - 지표(정확도, 정밀도, 재현율, 손실 등) 선정
- 자동으로 수행되는 것
    - 시간을 요하는 반복적 작업
    - 지표에 부합하는 최적의 모델을 선택하는 것

7. Azure Machine Learning
    - 워크스페이스 : Azure Portal
    - 기계학습모델 툴 : Machine Learning Designer

8. ボット機能を組み込んだWebチャット
- レストランの予約を自動的に取る
- コンサートチケットを購入する
- 銀行がユーザーからの営業時間などの質問に自動的に対応する
- 주의 : 소셜 미디에어 관한 기능은 포함하지 않는다

9. フューショット(Few-shot) プロンプティング
- 소수의 예시를 제시해 모델이 원하는 방식으로 응답하게 하는 방법

10. Azure AI 音声（Speech）
- 음성->문자(Speech-To-Text), 문자->음성(Text-To-Speech) 둘 다 가능

11. 멀티 클래스 vs 멀티 라벨
- 멀티 클래스 분류 : 各画像を必ず1つのクラスにだけ分類する(「犬」または「猫」)
- 멀티 라벨 분류 : 1枚の画像に複数のラベルを付与(「屋外」「犬」「夜間」)

12. しきい値
- 무턱값, 임계값, 경계값
- しきい値を高く設定 : 판정이 엄격해져 오검지 감소
- しきい値を低く設定 : 판정이 느슨해져 오검지 증가

13. Azure AI Language vs Azure AI Bot Service
- AI Language = 챗봇이 사람 말을 이해하는 두뇌
- Bot Service = 챗봇을 만드는 틀 (단독으로 쓰면 반복적이고 기계적인 대답만 할 가능성이 큼)

14. 평가데이터 vs 테스트데이터
- 평가데이터 : 튜닝용
- 테스트데이터 : 최종적인 평가용
