1. Git을 사용하는 목적
    - 파일의 버전을 관리하기 위해
    - 복수의 사람이 편집하기 위해

2. 버전관리의 이점
    - 파일의 버전을 기록하기 때문에 최신정보를 알 수 있다
    - 이전의 상태로 되돌릴 수 있다
    - 파일을 누가 편집했는지 알 수 있게된다.
    
3. Git의 역사
    - 리눅스의 커널개발에 사용하고있던 버전관리 시스템을 쓸 수 없게되어 Git을 사용하게 됨

4. Github

    - 다른 개발자들과 콜라보레이션 가능(풀 리퀘스트로 복수인원 개발)
